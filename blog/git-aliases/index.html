Git Aliases

Jul 12 2020

Introduction

This is the first post of “Workstation Setup” series. I will be covering how to setup various tools that I use in daily work in each episode. This series is inspired by the fact that I’m currently migrating from old computer to a new one at work. This episode will cover how to set up git aliases. In case you are not familiar with git, you should visit the official web page.

Aliasing

Aliases are a great way to boost your productivity. You can serve many keystrokes by optimizing the commands that you use most often. We should tell program aliases from shell aliases. We will cover program (ie. git) aliases in this post. Shell (ie. bash) aliases will be covered in another post.

Getting started

When it comes to git aliases we must start with the documentation page about git aliases. We will use git config tool heavily in this article. But don’t worry, you don’t have to focus on this a lot. I will cover git config in one of the upcoming posts. It is worth mentioning that you can not only create aliases for existing commands but you can also invent commands that you think should exist in git. On the aforementioned web page you can find four aliases. I highly recommend them as I use them a couple of times per day.

git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.ci commit
git config --global alias.st status

After applying changes you can type git st instead of git status. As you can see, this can save you a few keystrokes. Moreover, a good alias may prevent you from making a typo in a long command.

The following alias has been most helpful for me. I use it rarely when compared to previous ones. But when I have to use it I have to google it and it costs a lot of time. This is an example of alias which creates a custom command in git.

git config --global alias.unstage 'reset HEAD --'

You can use it in the following way git unstage someFile.

More advanced aliases

In order to add more advanced aliases it is better to edit the git config instead of running git config --global alias commands because you may encounter various shell escape issues. Git config file is in your home directory. You can use your favourite editor to check ~/.gitconfig file. Then navigate to [alias] label and start typing. If you added at least one of the aliases above, then your config file will contain that label.

Git aliases can run shell commands. Such aliases start with !.

Popular examples

Logging (as graph):#

git config --global alias.log-graph 'log --graph --all  --decorate --oneline'

Naming#

  • top name (in most cases directory name)
git config --global alias.top-name 'rev-parse --show-toplevel'
  • branch name
git config --global alias.branch-name 'rev-parse --abbrev-ref HEAD'
  • upstream name (edit git config directly)
upstream-name = !"git for-each-ref --format='%(upstream:short)' $(git symbolic-ref -q HEAD)"

Pruning stale branches (edit git config directly)#

prune-all = !git remote | xargs -n 1 git remote prune

Extreme optimisations

There are developers that want to optimise as much possible. Therefore they often alias git in their shell to alias g=git. Moreover, they utilise one letter aliases to type their most commonly used git commands as fast as possible:

# commit
c = "!git add -A && git commit -m "

# amend
a = !git add -A && git commit --amend --no-edit

# push
p = !git push origin $(git rev-parse --abbrev-ref HEAD)

# fetch changes
f = !git fetch --all && git rebase origin/master

# create new branch
n = !git checkout -b

Conclusion

Git aliasing is a very powerful tool. It allows you to fully customise your git configuration. There are countless resources on the web that you can use to make yourself more productive. Remember that the only limit is your imagination.
