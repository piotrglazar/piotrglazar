Java 14 Records

Sep 23 2020

https://dzone.com/articles/a-first-look-at-records-in-java-14

Introduction

Java 14 brings in another cool feature - Records. They are designed to represent only the state, without additional boilerplate. This feature is in preview, so you have to explicitly enable it in your project. You can check my previous post to see how to do it in Gradle. Alternatively, you can take a look at full example here.

How does a record look like

Take a look at the following piece of code:

public record Point(int x, int y) {}

Under the hood we could find that:

  • This class is final.
  • It extends java.lang.Record.
  • It has two private final fields x and y.
  • It has a public constructor Point(int, int).
  • It has two getters x() and y().
  • It has toString, equals and hashCode methods.

Additional methods and fields

It is not possible to add instance fields to records. However, it is possible to add static fields. Static and class (instance methods) can be added as well. The syntax is the same as for regular Java classes. Moreover, one can override a default constructor to add extra behaviour:

public record Range(int low, int high) {

    public Range {
        if (low > high) {
            throw new InvalidRangeBoundariesException(low, high);
        }
    }
}

Conclusion

Records allow programmers to implement data classes properly. The code is far less verbose when compared to regular classes. For further reading you can check the official page.
